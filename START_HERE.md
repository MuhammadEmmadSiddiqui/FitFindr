## 🚀 How to Use Your Application

### Quick Start (2 minutes)

```bash
# 1. Setup
python setup.py

# 2. Add your API keys to .env
# GROQ_API_KEYS=your_key_here

# 3. Run application
streamlit run streamlit_app.py

# Or with Docker
docker-compose up -d
```

### Access Points

- **Web UI**: http://localhost:8501
- **API**: http://localhost:8000
- **API Docs**: http://localhost:8000/docs
- **API ReDoc**: http://localhost:8000/redoc

## 💡 Interview Questions & Answers

**Q: Tell me about a project you're proud of.**

"I built FitFindr, an AI-powered resume screening platform. It uses BERT embeddings to calculate semantic similarity between job descriptions and resumes, then uses an LLM to extract structured information. I designed it with a microservices architecture - a Streamlit frontend, FastAPI backend, and SQLAlchemy database layer. The entire stack is containerized with Docker and has automated CI/CD. It reduced resume screening time by 90% and demonstrates my full-stack development skills."

**Q: How did you ensure code quality?**

"I implemented comprehensive unit testing with pytest, achieving over 60% coverage. I structured the code following clean architecture principles with clear separation between API, services, and data layers. I also set up CI/CD with GitHub Actions to run tests automatically on every commit."

**Q: How would you scale this application?**

"Several approaches: 1) Add Redis caching for embeddings, 2) Implement horizontal scaling with load balancing, 3) Move to PostgreSQL for better concurrent access, 4) Use message queues for async processing, 5) Deploy on Kubernetes for auto-scaling. The current architecture supports all these enhancements."

**Q: What challenges did you face?**

"API rate limiting was a challenge, so I implemented automatic key rotation. Processing large files required memory optimization, so I used streaming and batch processing. For production deployment, I used multi-stage Docker builds to reduce image size by 60%."
